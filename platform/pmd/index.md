# 量潮写作云 - 项目管理文档

## 当前阶段：核心链路打通（v0.1.0）

场景：CEO 助理理解 CEO 叙事风格，核心链路为 分析 → 理解 → 积累。

### 已完成

- 场景定义：BRD/PRD 完成，CEO 助理角色确立
- 服务端搭建：FastAPI 应用，`POST /review` 端点，规则驱动的段落叙事标签（起承转合）+ 风格对比
- 客户端搭建：Flutter Studio，与 API 联调通过
- 测试覆盖：无风格积累、有风格积累、风格演进三场景验证
- 规则引擎：good/bad 文章对比逻辑可产出可读建议

### 进行中

1. **服务端接入 DeepSeek API**
   - 状态：服务端和客户端已完成，风格积累与对比目前基于规则（`review.py` 硬编码判断），需要接入 DeepSeek API 替代或增强分析能力
   - 目标：用 DeepSeek 分析段落叙事结构 + 对比好文章风格，产出质量高于规则引擎
   - 验收：接入后对同一篇文章的 review 结果比规则引擎更准确、更具洞察

### 待完成

1. **后端部署到阿里云 FaaS**
   - 接入阿里云 OSS 作为后端对象存储（文章 + 分析结果持久化）
   - 打包 FastAPI 应用为函数计算部署包，配置触发器与路由

2. **前端部署到对象存储**
   - 构建 Flutter Web 产物，上传至阿里云 OSS
   - 配置 CDN 加速与自定义域名

### 待验证

1. **AI 校验价值验证**
   - 状态：接入 DeepSeek + 部署后验证。验证"给 AI 一篇陌生文章让它挑问题，结果是否有用"
   - 验收：拿一篇已发文章测 AI → 评估问题是否有用 → 判定是否形成固定流程
